# Migration Assessor

Analyze TypeScript, JavaScript, and Rust projects for migration to another language. Produces per-module compatibility scores, dependency graphs, symbol-level analysis, incremental diff tracking, and real-time target-project alignment to guide and verify incremental migration.

## Features

- **Symbol Extraction** — parses source code (via `oxc_parser` for TypeScript/JS, `syn` for Rust) to extract every exported/imported symbol with its type signature, visibility, and API contract
- **Dependency Graph** — builds a full directed graph of module-level dependencies, detects cycles, and computes topological layers for migration ordering
- **Compatibility Matrix** — maps each external dependency to its best-match equivalent in the target ecosystem, scoring compatibility as `Full`, `Partial`, `None`, or `Unknown`
- **Migration Scoring** — ranks every module by migration readiness across 5 weighted factors: in-degree (30%), complexity (25%), external compatibility (20%), cycle count (15%), test coverage (10%). Effort labels: trivial (≥70), moderate (≥50), heavy (≥30), rewrite (<30)
- **Diff Analysis** — AST-based incremental diff between git versions (tags/commits), detecting symbol additions, removals, renames, modifications, and breaking changes with propagation impact
- **Interface Boundaries** — computes dependency layers (topological zones) and identifies cross-layer "uncut surfaces" that must be addressed for incremental migration
- **Real-Time Target Alignment** — reads the target project on-the-fly (no database), extracts its symbols, and matches source changes to target locations using name translation, fuzzy matching, and signature comparison
- **Verify** — compares source diff symbol changes against uncommitted target changes, reporting coverage percentage (PASS/FAIL)
- **Check Updates** — compares analyzed commit against HEAD of the source repo, highlighting changed files, dependency changes, and risk tags

## Installation

```bash
cargo install migration-analyze
```

Or build from source:

```bash
git clone https://github.com/NLazyCat/migration-assessor.git
cd migration-assessor
cargo build --release -p migration-analyze
```

The binary is at `target/release/migration-analyze`.

## Quick Start

```bash
# 1. Scaffold a new migration project
migration-analyze init my-project
cd my-project

# 2. Edit migration.toml — set source_repo, source_lang, target_language
#    (source_lang is auto-detected from package.json or Cargo.toml)

# 3. Run full analysis
migration-analyze analyze

# 4. View terminal summary
migration-analyze summary

# 5. Incremental diff against latest version
migration-analyze diff --auto

# 6. Verify target changes match the source diff
migration-analyze verify --new-version v2.0.0
```

## CLI Commands

| Command | Description |
|---|---|
| `init <name>` | Create a new migration project with a `migration.toml` scaffold |
| `analyze` | Run full analysis: clone/find source, extract symbols, build graph, compute scores & compatibility, generate report |
| `summary` | Print a terminal summary of the latest analysis results (text or JSON) |
| `diff` | Run AST-based incremental diff between two git versions (`--new-version` or `--auto`). Resolves target-side symbols via alignment. Writes dated JSON + `latest.json` + `affected-files.json` |
| `verify` | Compare source diff against uncommitted target-project changes; updates `source_version` in `migration.toml` on PASS |
| `boundaries` | Generate an interface boundary report: topological layers + uncut cross-layer surfaces (text/JSON/HTML) |
| `check-updates` | Check the source repo for changes since the last analyzed commit; reports file changes, dependency diffs, and risk tags |

## Configuration

Projects are configured with a `migration.toml` file (auto-generated by `init`):

```toml
[project]
source_repo = "https://github.com/user/my-project.git"
source_lang = "typescript"
target_language = "rust"

# Target project path — enables real-time symbol alignment during diff
# target = "../my-rust-project"

# File glob patterns to ignore / exclude
ignore = ["**/*.test.ts"]
exclude = ["src/generated/**"]

[skip]
framework = true

[scoring.weights]
in_degree = 30
complexity = 25
compatibility = 20
cycles = 15
tests = 10

[mapping]
# Override file path mappings between source and target
# override_list = [{ from = "src/utils.ts", to = "new/utils.rs" }]
```

Source language is auto-detected from `package.json` (TS/JS) or `Cargo.toml` (Rust). If both exist, explicit `source_lang` is required.

## Output Structure

Running `analyze` produces `{repo}-migration/report/`:

```
report/
├── manifest.json              # Analysis metadata (source info, analyzed commit)
├── project.json               # Project-level overview
├── overview.json              # Aggregate statistics
├── scores.json                # Per-module readiness scores
├── errors.json                # Analysis errors (if any)
├── index.html                 # Interactive HTML report
├── graph/
│   ├── nodes.json             # Dependency graph nodes
│   ├── edges.json             # Dependency graph edges
│   └── cycles.json            # Detected cycles
├── symbols/{module}/
│   ├── symbols.json           # Extracted symbols per module
│   └── api.json               # API contract (exports)
├── references/
│   ├── forward/{file}.json    # Forward reference index
│   └── reverse/{file}.json    # Reverse reference index
├── external/
│   ├── packages.json          # Resolved dependencies
│   ├── compatibility.json     # Dependency compatibility matrix
│   └── recommendations.json   # Migration recommendations per dep
├── boundaries/
│   ├── layers.json            # Topological layers
│   └── uncut-surfaces.json    # Cross-layer interface dependencies
├── updates/                   # Generated by check-updates
│   ├── diff_overview.json
│   ├── changed_files.json
│   ├── dep_changes.json
│   └── summary.json
└── diffs/                     # Generated by diff
    ├── diff-{date}.json
    ├── latest.json
    └── affected-files.json
```

## Architecture

**migration-assessor** is a Cargo workspace with two crates:

- **[`core/`](core/)** — library crate containing all analysis logic: symbol extraction (language-specific parsers), dependency graph construction, reference tracking, compatibility matrix, scoring engine, diff engine, align module (real-time symbol matching), and verify module. Uses `oxc_parser` for TypeScript/JavaScript and `syn` for Rust.
- **[`migration-analyze/`](migration-analyze/)** — CLI binary built with `clap`, dispatching to 7 subcommands.

Language support is modular via a `LanguageRegistry` — each language (TypeScript, JavaScript, Rust) provides its own parser, symbol extractor, and naming conventions.

### Language Support

| Language | Parser | Status |
|---|---|---|
| TypeScript | `oxc_parser` | Full |
| JavaScript | `oxc_parser` | Full |
| Rust | `syn` | Full |
